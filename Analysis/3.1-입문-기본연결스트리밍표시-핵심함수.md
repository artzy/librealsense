## rs-hello-realsense

**소스:** `examples/hello-realsense/rs-hello-realsense.cpp`  
**중요 기능:** Pipeline으로 depth 스트림 시작, 화면 중앙 거리 콘솔 출력

| 함수 / API | 설명 |
|------------|------|
| `rs2::cli(...).process(argc, argv)` | CLI 인자·로그 설정 파싱 (`common/cli.h`) |
| `rs2::pipeline(settings.dump())` | JSON 설정을 넘겨 Pipeline 생성 |
| `pipeline.start()` | 기본 권장 구성으로 스트리밍 시작 |
| `pipeline.wait_for_frames()` | 다음 synchronized frameset 대기 |
| `frames.get_depth_frame()` | frameset에서 depth 프레임 추출 |
| `depth.get_width()`, `get_height()` | 프레임 해상도 |
| `depth.get_distance(x, y)` | 픽셀 (x,y)의 거리(미터) — 내부적으로 deproject·intrinsics 사용 |

## rs-distance (C)

**소스:** `examples/C/distance/rs-distance.c`  
**중요 기능:** hello-realsense와 동일 로직의 C API 버전

| 함수 / API | 설명 |
|------------|------|
| `rs2_create_context()` | RealSense 컨텍스트 생성 |
| `rs2_query_devices()` | 연결 장치 목록 조회 |
| `rs2_create_pipeline()`, `rs2_create_config()` | Pipeline·Config 생성 |
| `rs2_config_enable_stream()` | depth 스트림(640, Z16, 30fps) 명시 |
| `rs2_pipeline_start_with_config()` | 설정으로 스트리밍 시작 |
| `rs2_pipeline_wait_for_frames()` | frameset 대기 |
| `rs2_embedded_frames_count()`, `rs2_extract_frame()` | frameset 내 프레임 순회 |
| `rs2_is_frame_extendable_to(..., RS2_EXTENSION_DEPTH_FRAME)` | depth 프레임 여부 확인 |
| `rs2_depth_frame_get_distance()` | C API 거리 조회 (미터) |
| `rs2_release_frame()` | 프레임 핸들 해제 |

| `app.show(data)` | frameset을 GL 텍스처로 분할·타일 표시 |

## rs-pointcloud

**소스:** `examples/pointcloud/rs-pointcloud.cpp`  
**중요 기능:** color 텍스처가 매핑된 3D 포인트클라우드

| 함수 / API | 설명 |
|------------|------|
| `rs2::pointcloud pc` | depth → 3D points·texture 좌표 계산 |
| `frames.get_color_frame()` / `get_infrared_frame()` | 텍스처 소스 (RGB 없으면 IR) |
| `pc.map_to(color)` | 텍스처 프레임과 depth 정합 기준 설정 |
| `pc.calculate(depth)` | depth 프레임에서 `rs2::points` 생성 |
| `app_state.tex.upload(color)` | color를 GL 텍스처 업로드 |
| `draw_pointcloud()` | OpenGL로 points·texture 렌더 (`example.hpp`) |
| `register_glfw_callbacks()` | 마우스로 뷰 회전·팬 |

## rs-multicam

**소스:** `examples/multicam/rs-multicam.cpp`  
**중요 기능:** 여러 카메라 depth 동시 표시

| 함수 / API | 설명 |
|------------|------|
| `rs2::context ctx` | 다중 장치 관리 컨텍스트 |
| `ctx.query_devices()` | 연결된 모든 장치 |
| `dev.get_info(RS2_CAMERA_INFO_SERIAL_NUMBER)` | 시리얼로 장치 식별 |
| `cfg.enable_device(serial)` | 특정 장치만 Pipeline에 바인딩 |
| `pipe.poll_for_frames(&fs)` | 논블로킹 프레임 폴링 (다중 pipe 루프) |
| `rs2::sensor_from_frame(frame)` | 프레임 출처 센서·시리얼 역추적 |
| `colorizers[serial].process(frame)` | 장치별 colorizer 적용 |
| `app.show(render_frames)` | unique_id 맵으로 모자이크 표시 |

## rs-on-chip-calib

**소스:** `examples/on-chip-calib/rs-on-chip-calib.cpp`  
**중요 기능:** 온칩 자동 캘리브레이션(OCC)

| 함수 / API | 설명 |
|------------|------|
| `cfg.enable_stream(RS2_STREAM_DEPTH, 256, 144, Z16, 90)` | OCC용 저해상도·고FPS depth |
| `prof.get_device()` | Pipeline이 사용 중인 장치 |
| `dev.as<rs2::auto_calibrated_device>()` | 자동 캘리브레이션 인터페이스 |
| `cal_dev.run_on_chip_calibration(json, &health, progress_cb, timeout)` | JSON 설정으로 OCC 실행·진행률 콜백 |
| `cal_dev.set_calibration_table(res)` | 런타임 캘리브 테이블 적용 |
| `cal_dev.write_calibration()` | 플래시에 캘리브 결과 영구 저장 |
